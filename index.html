Query Runtime Analysis

    Made with:

Analyze the benchmark performance of different queries with and without indices and different database settings like Uninformed: undefined the primary and foreign keys of all tables and disabled the creation of auto-indexing, Self-optimized: (re)defined the primary and foreign keys of all tables and enabled the creation of auto-indexing, and User-optimized: create indices that would optimize the performance of the query load. The idea of using these scenarios is to enable one to investigate how varied the performance of a DBMS can be depending on how one sets it up.
